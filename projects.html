Leadme Mobile App

The Leadme mobile application provides offline functionality during trade shows and conditions with no internet connection. In the app, a user can view and manage incoming and previously captured leads. The app is built for Android, using Apache Cordova.

The app uses Cordova's Contacts Plugin to interact with an Android phone's local contact list. A feature is provided to allow users to write all of their Leadme leads into their phone's contacts list. The opposite function is also provided. A user can import leads from their contacts list as well. If an imported contact has a Leadme account, then the contact will be added to the user's leads.

The app uses the Cordova Pebblekit Plugin to communicate with Pebble's mobile app and a pebble watch. This interfacing is set up according to the process outlined in this tutorial by Pebble. When online, any handshakes that take place are immediately detected, sent to Leadme's server and evaluated. Any resulting connections are returned to the app immediately, and new lead information is updated. However, when a mobile user is offline, all potential handshake data is stored locally on the phone. This data is automatically processed when an internet connection is detected. Internet connection detection is based on Cordova's Network Information Plugin.

The graphical elements of the app are based on the free Minimum Admin template provided by Bootstrap Zero. In-app notifications are implemented using jGrowl.

Leadme Hardware

Leadme hardware is the wearable technology at the heart of Leadme's product. It is responsible for recognizing when two individuals shake hands and initiating the contact exchange. The hardware is a custom wristband built using Adafruit parts and written using Arduino Processing. It is currently a work in progress.

The wristband pairs with the Leadme Mobile app through Bluetooth and tracks wrist movement using an accelerometer. When it recognizes a handshake gesture it sends data to the mobile app which makes the connection to the other user on our server.

In addition to the custom wristband, there is a Leadme Demo application written for Pebble in C that provides very rudimentary gesture recognition for the purpose of Leadme product demonstrations.

Cents

Cents is a Laravel-based web application that provides online transactions through an integration with Stripe that has been developed by the Sigma team. The application adds a streamlined usability on top of Stripe’s transaction technology and adds functionality that supports group payment scenarios. We have implemented campaign creation and secure campaign sharing through campaign invite links. Every payment is tracked by the system as well as Stripe’s, and payment information can be accessed quickly by campaign owners and participants in simple roll-ups. Campaign information can be quickly communicated through campaign-specific photos and descriptions specified by campaign creators. Cents also supports transactions between pairs of users and options for capping contributors, specifying per-person charges, or allowing users to pay what they want. We have setup our own secure server through Vultr’s high performance SSD cloud service. A fully scalable virtual private server hosts the application which is based on tried-and-true technologies including scalable, encrypted data storage in PostgreSQL, interactive web design done in JQuery and secure network management using SSL.

Recreation CRM

Recreation CRM is a Customer Relationship Management web app co-written with 3 other developers. It is written to integrate with a suite of other applications that together manage all of the businesses functional requirements. There is a portal available at cs490-crm.herokuapp.com that provides a graphical overview of the company as a whole and individualized metrics and suggestions based on the RFM model for each customer. There is also an API available at /docs that can be used to add customers, purchases, promotions, events, customer interactions and customer information to our database as well as request analytics information. The app written in Python and is hosted through and run inside a Heroku dyno.

File Transfer Server

A TCP server written in Python using the socket interface, runnable in Unix. Clients connect and specify whether they would like to upload or download a file, as well as an alphanumeric identifier. The server opens a connection and blocks until a matching uploader/downloader is found, they initiates the file transfer. The server does not store any files.

The server is multithreaded. The main thread loops forever, blocking on accept until a connection is present. On a new connection, it starts a new thread for that specific connection. The threads all share access to three synchronized lists which track sockets (both waiting and transferring) as well as keys and are protected by a lock. First, the lists are checked for an opposite match (P/G), and if there is one the socket is placed in the transferring list and the transfer begins. A DONE event is also cleared at the beginning of a thread being added. If the key is not in the lists, it is added to them. At the end, the thread checks if all the lists are empty and if so it sets the DONE event so that the server can end if a client calls F.

Activity Tracker

A new type of activity tracker, designed to be used in conjunction with a keychain/phone case type device powered by Arduino. It is currently written entirely in Javascript with an HTML/CSS front-end. It is going to be accessible via a web app and a mobile app.

Users add buttons via the settings page for each activity that they do in a day, such that the categories cover the whole day's activities (eg. sleep, travel, gym, schoolwork, cooking/eating, leisure). They can also set buttons that simply "check off" a task, as opposed to an ongoing activity (useful for things one does every day and wants to track, such as "stretch" or "drink water").

The home screen of the app is just these buttons, and the user simply presses the button of the activity they are beginning. The previous activity is automatically ended and stored (users are able to edit these via the settings page), unless the button is a "check task" button. In place of opening the app, users will be able to simply press a button on a keychain/phone case (this will have a number of multi-coloured buttons that each represent a different task).

Users can then view how they spent their days/weeks in an easy to read graph format so that they can use that information to plan their day better and be more productive. It will offer different views such as a dashboard, total time at a specific activity, past week, past month etc.

This is currently a work in progress.
